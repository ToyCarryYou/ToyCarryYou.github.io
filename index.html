1.DNSLOG的函数解析

LOAD_FILE 读取文件的函数
读取文件并返回文件内容为字符串。要使用此函数，文件必须位于服务器主机上，必须指定完整路径的文件，而且必须有FILE权限，该文件所有字节可读，但文件内容必须小于max_allowed_packer（限制server接受的数据包大小的函数，默认1MB）。如果该文件不存在或无法读取，因为前面的条件之一不满足，函数返回NULL.
在cmd中读取文件
select load_file(‘c:/1.txt);
成功回显文件内容
只能读取绝对路径，其实可以读取远程文件
当我们共享了一个文件出去，后可看到文件路径发生改变。但处于同一个内网时，可以读取到同一网络下的文件。
\\的意义
\是转义字符很明显这里是转义两个反斜杠\（两个字符串的反斜杠）
这里引入UNC路径概念
UNC路径
通用命名规则（也称通用命名规范，通用命名约定）
（1）什么是UNC路径
UNC路径就是类似\softer这样形式的网络路径。他符合\servername\shearename格式，其中servername是服务器名，sharename是共享资源的名称
目录或文件的UNC名称可以包括共享名称下的目录路径，格式为：\servernane\diretory\filename
（2）例如softer计算机的名为it168的共享文件夹，用UNC表示就是\softer\it168我们熟悉的命令行访问法访问网上邻居，实际上应该称作UNC路径访问法
当我们在cmd中输入
select load_file(‘////baa.baidu.com//que’);
页面经过一定时间回显NULL
说明它确实会去访问这个域名而不是不可执行
此时会对dns发送请求，然后DNS将域名解析成IP返回给你
当我们自己搭建了一台DNS服务器时，如果访问某个网页，则会在服务器里留下记录。他人的访问也相同。由此，我们可以做一个字符串拼接

优先执行小括号语句，即子查询语句
则会将查询语句结果拼接入域名
直接在服务器查看访问记录即可得到查询结果
由此可使得盲注变为显错注入
一句话概括：通过LOAD_FILE函数读取UNC路径的远程共享文件，拼接函数将想要获得的东西做一个子查询，然后拼接到域名中，通过DNS记录的日志可直接查看。
可以通过DNS服务器平台实现此功能
http://ceye.io免费dnslog平台

通过平台给你的域名，当它被访问则会被记录

1.偏移注入使用场景

并不仅仅用于accssess,sql也可以使用
在sql注入的时候会遇到一些无法查询列名的问题，比如系统数据库带的权限不够而无法访问系统自带库
accsess不存在系统自带库，则无法知道字段名
联合查询必须字段相同
当猜到表名无法猜到字段的情况下，我们可以使用偏移注入来查询那张表里面的数据
像sqlmap之类的工具其实是爆破字段的名字，但是如果字段名字比较奇葩，例如：H5scker_Passwd之类的就无可奈何了
工具爆破即为穷举
引入知识
table_name.* 实际上是指表名.* 就是通配符
数据库中表名.
则为那个表中所有字段
在mysql中
select * from admin = select id,a,b from admin
=select admin.* from admin
将下列句子尝试改为
select * from admin where id=1 and 1=2 union select 1,2,3,4,5admin.* from admin
回显报错
因为前面的数字也为字段
我们将前面的数字删除
select 1,2 admin.* from
将所有数据成功输出（尽管不知道字段名）
偏移注入条件为：第一张表的字段比第二张表多（越多越好）

2.偏移注入步骤

（1）判断注入点
（2）order by 判断字段数
（3）判断表名 可通过sqlmap爆破

宽字节注入

Posted on 2019-06-03
1.什么是GBK编码格式

一种多字符编码，使用双字节编码方案，GBK汉子占2字节，UTF-8占3字节

2.宽字节SQL注入的原理

宽字节SQL注入主要是源于程序员设置数据库编码非英文编码那么就有可能产生宽字节注入
例如说MySql的编码设置为了SET NAMES ‘gbk’或是SET character_set_client = gbk,这样配置引发编码转换从而导致的注入漏洞
宽字节注入就是PHP发送请求到MySql时使用了语句
SET NAMES ‘gbk’或是SET character_set_client=gbk进行了一次编码，但又由于一些不经意的字符集转换导致了宽字节的注入
函数引入
magic_quotes_gpc（魔术引导开关）
作用：当php的传参中有特殊字符就会在前面加转义字符“\”来做一定的过滤
我们发现\的编码是%5c，“運”的gbk编码为%df%5c
可以用%df吃掉%5c，让单引号闭合逃逸转义字符
ps：只有国内网站使用gbk编码，国外网站基本不用考虑
解读编码为分段解读
防止宽字节注入：将数据库编码改为utf-8

注入实操

1.?id=1’
反斜杠将代码转义为字符串
2.在?id=1’后加上%df
将反斜杠吃掉，成功写入注入语句
有可能遇到的问题
指定系统自带库的表名库名时单引号被转义
如果继续拼接%df，则为
union select 1,2,運’a運’
数据库无法理解则报错
可以将%df和查询语句写成16进制
sql语句支持16进制的输入（加上0*，以区分字符串与进制语法）
